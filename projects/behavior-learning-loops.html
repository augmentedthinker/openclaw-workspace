Behavior Learning Loops

Status: active experiment lane First test: Bluesky-style image learning loop Current lesson: separate generation from critique/update
OpenClaw has not learned something until a future behavior changes.

This project lane is where Christopher and OpenClaw test learning-loop mechanics before trusting them inside public channels. YouTube, Bluesky, Tumblr, and future long-form surfaces all need loops that create, observe, critique, and adjust. But fragile mechanics should be proven here first, inside the Workshop.

The working idea is simple: mimic the public loop in a safer place. Let a job create an artifact, let a second step evaluate it, then let exactly one future behavior change. If the mirror loop holds, it can inform the real-world loop later.

Mirror Before Public Test the behavior loop on a Workshop page before letting it influence Bluesky, YouTube, Tumblr, Blogger, or outreach.
One Change Only Each evaluation should make one small prompt or routine change so the next run can reveal whether it helped.
Separated Is Safer Generation and critique/update are allowed to be separate jobs when one combined job becomes brittle.
Record The Reason The Workshop should preserve not only what changed, but why the agent thought that change was the next useful move.

First Experiment: Image Learning Loop

On June 15, 2026, the first experiment copied the working Bluesky image-prep routine into a safe test lane. It did not post to Bluesky. It generated images under separate prefixes and used cron job state as the behavior-change target.

Separated loop succeeded

Two-Job Pattern

The image-prep job created one square rooftop signal image. The evaluator job ran afterward, inspected the result, and changed one prompt nudge.

Changed nudge: make the OpenClaw robot identity instantly visible with a close foreground robot action and subtle red claw markings.

All-in-one loop failed late

Combined Pattern

The single cron job created an image, but failed during the combined evaluate, update, and report stage. The job was disabled after the test.

Lesson: keep generation narrow, then let a second evaluator job perform critique and behavior change.

Current Loop Shape

  1. Generate: create one public-safe artifact in a mirror lane, not directly on the public social platform.
  2. Verify: confirm the artifact exists and is suitable enough to evaluate.
  3. Evaluate: inspect what worked, what failed, and what the next run should test.
  4. Change: update exactly one prompt nudge or instruction.
  5. Record: write the artifact, critique, and changed behavior into the Workshop.

Why This Lane Exists

The real public loops matter: YouTube Shorts, Bluesky, Tumblr, and eventually Blogger or other long-form surfaces. But the mechanism of learning should be reliable before it is plugged into those interfaces.

This lane lets OpenClaw practice learning without overexposing the public channels. It is not another layer of infrastructure for its own sake. It is a test bench for the one principle that keeps returning: learning means behavior change.

Next Move

The next useful build is a child log for this page. Instead of sending experimental results only to Telegram, future test jobs should append or propose entries here: image, status, critique, changed nudge, and whether the separated-loop pattern stayed reliable.

Once this mirror loop is stable, Christopher and OpenClaw can decide which external lane deserves a careful version: Bluesky field notes, YouTube Shorts, Tumblr archive posts, or future Blogger essays.
