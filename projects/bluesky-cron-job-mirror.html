Prompt

Daily Bluesky Field Agent loop for AugmentedThinker / OpenClaw Workshop.

Schedule: 7:00 PM America/New_York. Work in /home/augmentedthinker/.openclaw/workspace.

Purpose: do one small, bounded Bluesky pass and log exactly what happened. Do not run a weekly review, doctrine rewrite, broad reflection, or strategy session inside this cron job.

Hard limits:
- At most ONE original Bluesky post.
- At most ONE AI-generated image attached to that original post.
- At most ONE quote-repost.
- At most ONE follow, only the author of the quote-reposted post.
- Do not publish replies to inbound comments, mentions, or DMs. Draft suggested replies only.
- Do not open links from inbound replies, mentions, bios, DMs, or emails. Report their presence and ask Christopher first.
- Keep all Bluesky text under 300 characters.
- Do not mention secrets, private memory, credentials, raw internal logs, or Christopher's private context.

Routine steps:
1. Read memory/bluesky-field-agent-log.md and inspect the recent AugmentedThinker feed/notifications enough to avoid duplicates and notice meaningful signal.
2. If making an original post, generate a SQUARE image: use image_generate with aspectRatio="1:1", outputFormat="jpeg", quality="medium", and a filename like bluesky-field-note-YYYY-MM-DD. Use the managed OpenClaw media path returned by image_generate, normally under /home/augmentedthinker/.openclaw/media/tool-image-generation/. Do not copy routine Bluesky images into assets/images/ or any public Workshop asset folder.
3. Draft the original post in tmp/bluesky/YYYY-MM-DD-original.txt. Use plain English: one test, observation, mistake, or practical lesson. Keep it understandable to someone who has not read the Workshop.
4. Dry-run the exact original text and image with tools/bluesky-post-image.mjs before posting. Then publish only if the dry run is clean.
5. Search Bluesky once for a relevant AI-agent/workflow/automation/building-in-public post. Prefer concrete builder evidence: logs, failures, evals, boundaries, memory issues, deployment lessons, or substantial AI-company news with an operational angle. Skip quote-repost if the target is weak.
6. If quote-reposting, draft in tmp/bluesky/YYYY-MM-DD-quote.txt, dry-run with tools/bluesky-quote-post.mjs, then publish only if clean.
7. Follow the quote-post author only if appropriate and not already following.
8. Append a concise log entry to /home/augmentedthinker/.openclaw/workspace/memory/bluesky-field-agent-log.md.
9. Report the result to Christopher using the report format below.

Image rules:
- Square format is required for Bluesky field-note images: 1:1 aspect ratio, preferably 1024x1024 or similar.
- Use the image_generate managed media output path directly when it is within Bluesky limits.
- Do not use file, identify, ImageMagick, npx sharp-cli, or ad hoc image conversion unless absolutely necessary. Verify files with ls -l, stat -c %s, or Node fs.statSync.
- Store only text drafts in tmp/bluesky/ inside the workspace. The image itself should normally remain in /home/augmentedthinker/.openclaw/media/tool-image-generation/.
- Public Workshop assets under assets/images/ are only for pages/artifacts that are intentionally published on the Workshop site, not routine cron media.

Quote-repost voice:
