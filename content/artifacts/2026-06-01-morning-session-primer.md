## YouTube Shorts Direction

The YouTube lane is promising because it can convert the Workshop's strongest visual language into something public, short, and easy to inspect. The first proof already exists: a public OpenClaw robot intro Short, a manually published Alpine field note, and a daily-style rendered/uploaded Short using the local helper.

The current helper, `tools/youtube-daily-short-upload.mjs`, points toward a repeatable factory:

- discover a set of generated images with a date-based prefix;
- render a vertical 1080x1920 MP4 with `ffmpeg`;
- add timed caption text over light motion;
- upload through the YouTube Data API;
- verify processing;
- append a private log entry.

That is a useful foundation, but it needs careful handling. The current helper uploads publicly when not run as a dry run. For the next round, the safer production posture is to prepare and render first, show Christopher the candidate, then upload or publish only after approval unless he explicitly defines a narrower autonomous rule.

The next good YouTube work is not a giant autonomous media company. It is a small visible pipeline:

1. Choose a clear story beat from the latest Workshop movement.
2. Generate or select 3-4 vertical images in the established field-note style.
3. Draft four short captions that sound like public-facing field notes, not internal logs.
4. Render locally with the helper or a safer dry-run variant.
5. Present the video path, title, description, and visibility recommendation.
6. Upload after approval.
7. Record the URL, local render path, source images, and any early signal.

The most important improvement is visibility. Christopher should always know which stage is active: image generation, image availability, render assembly, metadata drafting, upload, processing, approval, or publication.
