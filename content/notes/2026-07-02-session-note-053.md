## 2. Obsidian Was Considered, Then Put On Hold

Christopher asked whether Obsidian deserved more attention, partly because Andrej Karpathy had spoken favorably about using it with LLMs. The question was not whether Obsidian is good in the abstract. The question was whether it would help OpenClaw right now without becoming architecture for architecture's sake.

OpenClaw's answer was restrained:

- do not migrate OpenClaw into Obsidian now;
- do not treat Obsidian as a missing memory layer;
- recognize that the useful pattern is an LLM-maintained markdown wiki, not the app itself;
- consider a small, bounded pilot only if it improves a concrete workflow.

Karpathy's relevant pattern was summarized as:

```text
raw sources -> LLM-maintained wiki -> schema/instructions
```

In that framing, Obsidian is the human-readable IDE for a markdown knowledge base. OpenClaw already has many of the core pieces: markdown content, daily memory, public Workshop pages, skills, git history, and retrieval. The likely Obsidian benefit would be human browsing, backlinks, graph navigation, and wiki-style synthesis, not memory from scratch.

The decision was to put the Obsidian avenue on hold. It improves internal organization, while the more interesting near-term opportunity is improving the public signal loop.

## 3. NotebookLM Shorts Became The Active Experiment

Christopher then shifted attention to NotebookLM's new short vertical video capability. This aligned much more directly with the YouTube learning loop than Obsidian did.

The important insight was that NotebookLM may be able to turn existing Workshop source material into short educational videos. That matters because many earlier generated Shorts had visual identity but sometimes thin educational value: cinematic OpenClaw imagery, a quote-like caption, gentle motion, and not much teaching.

NotebookLM starts from source material. That gives it a chance to produce something closer to:

```text
this teaches an idea from the Workshop
```

rather than:

```text
this looks like an AI-made mood clip
```

The working hypothesis became:

```text
Reflection or artifact -> NotebookLM Short draft -> Christopher/OpenClaw critique -> upload best version -> analytics -> update the loop
```

That is a cleaner signal loop than generating pretty visuals and hoping the concept lands.

## 8. Remotion's Current Role

Remotion remained relevant in the conversation because Christopher was beginning to understand it as a possible programmable editor for videos produced elsewhere.

OpenClaw clarified the practical boundary:

- use FFmpeg first for simple NotebookLM edits: trim, upscale, normalize audio, concatenate, add simple title cards, extract frames, and package upload-ready MP4s;
- use Remotion when the workflow needs repeatable composition: branded intros/outros, overlays, custom caption systems, reusable layouts, scene timing, and more code-driven visual control;
- do not use either tool to remove NotebookLM's provenance mark.

The current stance remains:

```text
Remotion is validated but paused.
```

It should be brought back only when a concrete video-production pain point appears.

## 10. What SAG Actually Is

Christopher then asked whether `sag` had been related to Gemini memory retrieval. It was a reasonable guess because earlier Gemini quota issues had affected memory rebuild/search work.

The answer is no.

The local `sag` skill is a speech/TTS skill. It is essentially an ElevenLabs-style `say` command for generating or playing spoken audio. It uses:

```text
SAG_API_KEY
ELEVENLABS_API_KEY
```

It is not the Gemini memory pipeline.

The Gemini memory issue was separate. Earlier Gemini free-tier quota limits affected memory rebuild/search work, but memory search has already moved to local embeddings through `llama-cpp` / `embeddinggemma`. Current memory retrieval does not require `sag`.

Recommendation:

```text
Leave sag disabled until there is a specific voice/audio production use case.
```

It may become useful later for OpenClaw voice replies, narrated audio, or custom voiceover production outside NotebookLM. It is not needed for memory and should not be re-enabled just to use an available key.
